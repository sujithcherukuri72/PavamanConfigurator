### **5. Color Coding Reference**

#### **Event Severity Colors**

| Severity | Badge Background | Text Color | Use Case |
|----------|-----------------|------------|----------|
| **Info** | `#EEF2FF` (Light Blue) | `#3B82F6` (Blue) | Normal events, mode changes, status |
| **Warning** | `#FEF3C7` (Light Yellow) | `#F59E0B` (Orange) | GPS poor, high vibration, minor issues |
| **Error** | `#FEE2E2` (Light Red) | `#EF4444` (Red) | Battery low, sensor errors |
| **Critical** | `#FEE2E2` (Light Red) | `#DC2626` (Dark Red) | Crash, critical battery, major failure |

### **7. Expected Event Types**

Your log file should contain events like:

? **Info Events:**
- Log Start
- Armed
- Disarmed
- Mode Changed (e.g., Stabilize ? Loiter)
- GPS Acquired
- Takeoff Detected
- Landing Detected

?? **Warning Events:**
- GPS HDOP High (poor GPS quality)
- High Vibration
- EKF Warning
- RC Signal Quality Low

? **Error Events:**
- Battery Low
- GPS Lost
- RC Lost
- EKF Error
- Compass Variance
- Accelerometer Clipping

?? **Critical Events:**
- Battery Critical
- Crash Detected
- EKF Failsafe
- GPS Failsafe
- RC Failsafe

## ?? Tips for Best Results

### **For Events Tab:**
1. Load a flight log with actual flight activity (not just boot-up)
2. Keep all severity filters checked initially
3. Use search to find specific issues
4. Click events to see telemetry data at that moment

### **For Parameters Tab:**
1. Verify JSON file is at: `C:\Users\<YourUser>\Desktop\5\json1.json`
2. Use search to find parameter groups (e.g., "ATC", "BATT", "GPS")
3. Click parameters to see full details and options
4. Check enum parameters for valid value descriptions
